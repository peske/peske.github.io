Remarks
Joined widgets have a special behavior regarding the methods mentioned here. For details see "Remarks" section in WidgetCommonBase Constructor(WidgetCommonBase). Implemented here, methods BeforeLoopFirst, BeforeLoop and AfterLoop don't do anything. But they are implemented as virtual to allow inherited classes to override them to implement different logic in different stages of socket's lifecycle. Methods BeforeLoopFirst and BeforeLoop (as their name implies) are called before infinite loop starts. You can override them to implement any initialization logic that needs to be executed before the loop starts. The difference between these two methods is that BeforeLoopFirst is called only once during the instance's lifetime, after the instance is started for the very first time, and before the loop. On the other side, BeforeLoop method can be called multiple times - after every start. To remind you - the instance is started automatically after its connected / bound, and stopped automatically if disconnected / unbound. Since the instance can be connected / disconnected multiple times - it means that it can also be started / stopped multiple times. It is not too often scenario, but still possible. Just to be perfectly clear: after the very first start both these methods will be called, while in all subsequent starts only the second one will be called.

Method AfterLoop gets called after the main loop. It'll happen if the instance is stopped or disposed.

Another important thing to note is that all three methods mentioned here are called from the same thread that will run the loop (thus all message sending / receiving).
