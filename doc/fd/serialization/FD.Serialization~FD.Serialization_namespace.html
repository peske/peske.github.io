Inheritance Hierarchy
Classes
 ClassDescription
AnonymousSerializerBase Intended for serializer library developers. Base class that should be inherited by all classes that are meant for anonymous serialization.
DotNetCoreHelper Intended for IT Enlight employees. Helper class that unifies .NET Core and full .NET framework APIs.
Extensions  
FastActivator Intended for IT Enlight employees. Compiled-LINQ implementation that should be faster than System.Activator for creating instances of a specific type (according to this article).
FdSerializable Intended for serializer library developers. Base class from which all types that will be (de)serialized should inherit, if typed serializer mechanism is used. (For explanation about typed serializers check "Remarks" section in AnonymousSerializerBase.)
FdWrappingSerializable<T,TMust> Intended for serializer library developers. Base class used derived in cases when proxy pattern is used for message types.
RawSerializationInfo Class that contains options used during raw (de)serialization. Raw deserialization is explained in "Remarks" section of SerializationCentral.SerializeRaw.
ReadOnlyMaskedStream Class that wraps another stream, and exposes only its part.
ResetableNonSeekableStream Class that wraps non-seekable streams (i.e. network stream), and allows resetting to the initial position.
SerializationCentral This is the central class in FD.Serialization library. It is used for serialization, deserialization, creating an instances of message types, etc.
SerializationManagerBase Intended for serializer library developers. Every serializer library has to have one class that inherits from this base class. It's purpose is to describe the serializer and it's features.
SerializationOptions Implementation of ISerializationOptions
SerializationOptionsAttribute Attribute implementation of ISerializationOptions.
SerializationOptionsReadOnly  
SerializedContainer Class used internally for instantiating and deserializing FdSerializable instances. It has no any public members, so there's nothing to know about it except that types that are inheriting from FdSerializable have to implement non-public constructor that accepts one argument of this type.
Serializers Defines known serializers.
TypeBookBase This class is used as internal type address book. It enumerates all message types used across your solution, and defines their ID. Example usage is available at Example.
TypeIdentifier Intended for IT Enlight employees. Represents type identifier. The reason for introducing a new type for this purpose is in the fact that type IDs can be configured with a different lengths, still we need to have common way to use these.

IFdCopyable Interface that all types that will be (de)serialized should implement if multiple serialization mechanisms will be used.
IFdSerializable Empty interface (without any members) used for marking message types. Message types are types that will be (de)serialized.
ISerializationOptions Represents serialization options used. Corresponds to fd_serialization:serialization:options section from the configuration file.
