Remarks
Conversion to / from System.Byte array is performed in big endian layout. Big endian is selected because NetMQ library uses it for serializing System.Int32 and System.Int64 values. Since both are using big endian layout - they can be used interchangeably, meaning that you can use methods from FD.ZeroMQ library to send an integer value, and NetMQ library to receive the value, and vice versa. Here's an example: Note that interchangeability described stands only for System.Int32 and System.Int64 types. Besides these FD.ZeroMQ library supports more types that don't have the appropriate counterpart in NetMQ: System.Int16, System.UInt16, System.UInt32, System.UInt64, System.Double.
