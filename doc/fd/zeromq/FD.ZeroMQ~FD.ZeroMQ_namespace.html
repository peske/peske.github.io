Interfaces
 InterfaceDescription
IDealerSubscriber Defines special widget based on DEALER socket that is used as a subscriber. Widgets of this type have to be used with IRouterPublisher widgets on the other end.
IInstanceCreator Instance that implements this interface will be used for creating instances of different types.
IReceiver Represents ZeroMQ widgets that are receive-only (like SUB socket is).
IReceiveSender Represents receive-send ZeroMQ widget. It is a widget that operates in strict receive-send-receive-send-... manner (like REP socket).
IRouterPublisher Defines special widget based on ROUTER socket that is used as a publisher. Widgets of this type have to be used with IDealerSubscriber widgets on the other end.
ISender Represents send-only ZeroMQ widgets (like PUB socket).
ISenderAndReceiver Represents send-free-receive-free ZeroMQ widget.
ISendReceiver Represents send-receive ZeroMQ widget. It is a widget that operates in strict send-receive-send-receive-... manner (like REQ socket).
ISubscriber Defines widgets that wrap SUB socket and provide (un)subscribing functionality.
IWidget Defines connection / binding methods which are common for all ZeroMQ widgets.
IWidgetSocket<T> Extends IWidget and adds Socket property.
