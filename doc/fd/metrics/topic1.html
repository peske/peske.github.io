Parameters

reports
FD.Metrics.Reporters.MetricsReports instance to be configured.
kairosDbEndpoints
Enumeration of KairosDB REST endpoints.
tags
Tags that will be attached to all metrics. In KairosDB tags are specified as key-value pairs, and every metric has to have at least one tag defined. If this parameter is not specified default tag is used: host=<HOSTNAME>.
contextsIgnore
Allows filtering Metrics.NET contexts that will be saved. Contexts enumerated in this parameter, as well as their sub-contexts, will be ignored.
contextsInclude
Allows filtering Metrics.NET contexts that will be saved. If this parameter is specified, only contexts contained by this parameter (and their sub-contexts) will be saved.
interval
Reporting interval. Defaults to 1 second.
includeHealthStatus
Whether health status info should be saved in KairosDB or not. Defaults to true.
failBuffer
When KairosDbReport is unable to save the metrics (for example KairosDB database is unreachable), it will keep the metrics in memory and try to save them later. It is so-called fail buffer feature. This parameter specifies the maximal number of metrics in fail buffer (fail buffer capacity), and its purpose is to prevent memory overflow. Once fail buffer is full it will not accept additional metrics, and any metrics that is not successfully saved will be discarded. Defaults to 100,000.

Return Value

Configured FD.Metrics.Reporters.MetricsReports instance.
Example
