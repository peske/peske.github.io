FD.Metrics Library
Overview
Compiled (CHM) version of this documentation can be downloaded here.

Assemblies from FD.Metrics namespace are meant for creating and collecting application metrics data. They are based on Metrics.NET open source project which we've addopted to our needs.

What We've Changed?

Our primary goal was to create .NET Core version of the original project. It is accomplished by:

  • Breaking the original project into multiple projects based on functionality, as described in issue 132;
  • Creating .NET Core version of Metrics.NET core part.

While breaking the original project in multiple parts we've preserved only parts that are of interest for us:

  • FD.Metrics assembly which represents the core part of Metrics.NET, and which targets both full .NET framework and .NET Core;
  • FD.Metrics.Visualization assembly which contains some HTTP visualization and publishing features. FD.Metrics.Visualization targets only full .NET framework (.NET Core version is not created).

Some other parts of the original project, for example ElasticSearch, InfluxDB and Graphite reporting are not implemented at the moment.

Versioning

We've introduced convention that all the assemblies gotten by breaking the original project will have the same version as the original project. For example starting from Metrics.NET v0.3.7 we've created FD.Metrics v0.3.7 and FD.Metrics.Visualization v0.3.7.

Namespaces

We've renamed the original namespaces by adding "FD" prefix. For example namespace "Metrics" in the original project is "FD.Metrics" in the newly created assemblies.

What We've Created?

Since we are using KairosDB for storing the metrics we had to create FD.Metrics.KairosDB Assembly which saves Metrics.NET reports to KairosDB database. For details about usage see WithKairosDb Method.
