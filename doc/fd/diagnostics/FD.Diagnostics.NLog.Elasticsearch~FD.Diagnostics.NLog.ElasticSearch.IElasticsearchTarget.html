Remarks
When it comes to configuring ElasticSearch target for NLog, the overall configuration can be grouped into two groups:
  • ElasticSearch-related configuration (i.e. ElasticSearch server's URL, index to be used, document type, etc.). In a manner of speaking it is where-and-how to write. Configuration options from this group are represented by this interface, and are set as attributes of <target> configuration element, as shown below.
  • Log content configuration (i.e. what fields we want to include in the log entry to be saved). It is what to write configuration. For this purpose LayoutRenderer mechanism (embedded in NLog) is used, and desired fields are enumerated in NLog config file as children elements of <target> configuration element, as shown below.
The following snippet shows syntax of <target> configuration element: ]]> The code from the snippet is not configuration example, but syntax representation (attribute values used are not configuration values but configuration value types. As already said, <target> element attributes are represented by IElasticsearchTarget interface and its properties, so please check documentation of the properties for explanation.

The only attribute that is not exposed as IElasticsearchTarget interface's property is xsi:type. It represents the target type. FD.Diagnostics.NLog.ElasticSearch assembly offers two target types:

  • ElasticsearchTarget
  • ElasticsearchTargetEx
Configuration options explained here are the same for both mentioned target types.
